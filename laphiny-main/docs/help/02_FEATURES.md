## @ 路由

- 单聊默认发给唯一成员。
- 群聊无 @ 默认不自动发送，除非房间设置了默认协作模式。
- `@成员名` 指定一个或多个 Agent。
- `＠成员名` 支持中文输入法全角 at。
- `@all` 并行调用所有启用成员。
- `@all-seq` 按房间成员顺序接力调用。
- Agent 回复中行首 `@成员名 具体任务` 会创建委托任务并触发被委托 Agent。
- 委托有最大深度限制，避免无限循环。

## Soul-native 协作

- 协作时间线。
- 委托任务卡。
- 任务看板：待处理、处理中、已完成、阻塞 / 失败。
- 团队模板。
- 房间共识总结。
- 协作卡片自维护。
- Soul 关系图。
- 今日小队动态。

## 协作仪式

- `/council` 议会模式：多 Agent 独立观点，然后总结共识。
- `/redteam` 红队模式：找漏洞、风险、失败场景和修正方案。
- `/review` 审查模式：从不同能力视角审查方案、代码或交付物。
- `/retro` 复盘模式：总结进展、问题、贡献和下一步。

## 房间模式

- 工作室：适合项目讨论和产出。
- 议会：适合决策。
- 审查：适合风险和质量检查。
- 桌游：适合 RP。
- 日常：适合轻松聊天。

房间模式会影响默认协作方式、是否自动委托、RP 是否启用、提示词风格和 UI 状态条。
