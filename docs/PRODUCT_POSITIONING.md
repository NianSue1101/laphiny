# Laphiny 产品定位与迭代方向

## 当前定位

Laphiny 是 **Soul-native 的本地优先多 Agent 协作空间**：连接到用户已经拥有身份、人格、服务端记忆的 Hermes Agent，而不是在客户端临时拼装角色提示词。产品核心是让多个稳定 Agent 在同一房间里共享必要上下文、分工、沉淀决策，并由用户掌握方向与隐私边界。

它不是通用模型聚合器，也不是自动替用户执行外部动作的自治框架。它更像一间可持续经营的 AI 工作室：用户负责召集、授权和最终确认；Agent 在清晰可见的协作协议下完成讨论、委托与交付。

## 已具备的能力

- 多 Hermes 连接、单聊/群聊、各成员独立会话键与流式回复。
- 精确 `@` 路由、并行/接力协作、带质量门槛和深度限制的 Agent 委托。
- `/goal`、议会、红队、审查、复盘与 RP 等复用同一房间原语的协作流程。
- 目标计划、验收条件、复盘状态、任务看板、知识库、黑板、决策记录和待确认记忆胶囊。
- Android/Web/PWA、本地优先持久化、备份、可选 SQLite 同步、诊断脱敏与通知。
- 分页消息存储和按成员串行/跨成员并行的运行时，支持长记录与多 Agent 交错工作。

## 核心差异

1. **保留 Agent 身份**：Laphiny 只提供协作层，不覆盖 Hermes Soul。
2. **把协作变成可审计状态**：委托、计划、记忆、决策和任务不只停留在聊天气泡中。
3. **本地优先、用户确认**：敏感连接不内置；记忆与高影响操作保持显式确认。
4. **移动端可用的持续工作台**：不是只适合桌面的多栏面板，Android 也能处理长历史、并发回复和房间切换。

## 后续优先级

### 近期：把“可用”打磨成“可靠”

- 增加历史记录的显式“加载更早消息”和跨分页全文搜索，保留当前快速启动优势。
- 在任务看板展示委托输入、产物、验收和失败原因，支持重试/改派。
- 完善首次连接向导、隐私状态说明和同步冲突的可操作修复。

### 中期：把目标模式做成可控闭环

- 为目标增加预算/时间/轮次上限、显式暂停与调整记录。
- 根据 Agent 协作卡片、历史完成质量和任务类型推荐负责人，并始终允许用户改派。
- 让主 Agent 汇总每轮委托结果后生成可确认的最终交付与风险清单。

### 长期：形成 Soul-native Agent Studio

- 记忆胶囊的差异比较、版本回滚和来源追踪。
- Agent 文件产物中心与项目型房间模板。
- 面向自托管用户的同步、反馈、权限审计和团队协作能力，同时不牺牲本地优先原则。
