# Laphiny 阶段三长期目标

阶段三的目标是把 Laphiny 从“可用的多 Agent 聊天原型”推进到“可长期使用的私人多 Agent 工作台”。

## 阶段三完成标准

1. **日常管理完整**
   - 连接可以新增、编辑、停用、删除、测试健康状态。
   - 房间可以创建、重命名、删除、清空本地消息、重置 Hermes 会话。
   - 群聊成员可以新增、移除、启用/停用、修改别名。

2. **长期记录可用**
   - 支持跨房间搜索消息、作者、附件名。
   - 支持单房间 JSON / Markdown 导出。
   - 支持完整备份与合并恢复，覆盖连接、房间、消息、灵庭事件、同步配置和诊断日志。
   - 原生端大量长期记录使用文件存储，API Key / 同步密钥继续留在 SecureStore，避免把不断增长的聊天历史塞进 SecureStore。

3. **协作上下文稳定**
   - 群聊请求给 Agent 的共享记录必须带说话人标签。
   - 支持 @成员、@all 并行、@all-seq 接力。
   - Agent 回复中的行首 @ 委托应在单个 Agent 回复完成后立即触发，并有最大委托深度限制。
   - Agent 协作路由优先依据各自自我生成的公开协作卡片，而不是 Laphiny 硬编码人格。

4. **私有同步可维护**
   - 轻量同步后端可以同步快照和灵庭事件。
   - 客户端提供后端健康检查、手动推送、手动拉取、轮询事件。
   - 本地备份作为同步失败时的安全兜底。
   - 推送/拉取前支持同步差异预检，只读取远端快照并展示本地独有、远端独有、本地较新、远端较新和同 id 内容差异。

5. **部署和排障明确**
   - Web 构建统一使用 `npm run web:build`。
   - 关键操作失败时给出用户可读错误。
   - 保留最近 Hermes 请求、Agent 委托、连接测试、同步和备份恢复的诊断日志。
   - 可复制脱敏诊断包，便于远程排障时不泄露 API Key。
   - Web/PWA 支持安装提示、Service Worker 离线缓存、离线状态提示。
   - Android / 平板支持横竖屏和大屏可调整布局，诊断面板显示当前布局模式。
   - 保留 AGENTS.md 中的部署和协作规则。

## 当前迭代批次

已完成的阶段三底座：

- 全局消息搜索。
- 完整备份 / 合并恢复。
- 连接编辑 / 删除。
- 房间重命名 / 删除。
- 群成员别名、开关、加入、移除。
- 默认连接列表与 repository 默认保持一致，包含 Derux。
- 同步后端持久化 connection.profile、room.sessionIds 和 room.memberSessionKeys，避免跨设备协作卡片和 Hermes 会话上下文丢失。
- 诊断日志面板：记录 Hermes 请求、委托、连接测试、同步、备份恢复，并支持复制脱敏诊断包。
- 原生端长期记录存储迁移：rooms / messages / squareEvents / diagnosticLogs 改用 expo-file-system 持久化，保留旧 SecureStore 数据的一次性迁移。
- 诊断面板展示当前密钥存储后端、长期记录存储后端和消息体积，诊断包包含存储摘要。
- 同步差异预检：推送/拉取前可只读远端快照，展示连接、房间、消息、灵庭事件的本地/远端差异。
- Web/PWA 运行体验：安装提示、在线/离线状态提示、Service Worker 运行时缓存和离线兜底页。
- Android 大屏收口：允许横竖屏和 resizeableActivity，诊断面板显示平台、宽度和布局模式。

## 阶段三完成状态

阶段三已完成当前定义下的可长期使用闭环：日常管理、长期记录、协作上下文、私有同步、备份恢复、诊断排障、PWA 离线安装体验和大屏运行诊断都已具备。

后续如果继续增长，可以作为阶段四或专项优化推进：

- 消息量继续增长时，从 JSON 文件升级到 SQLite 分片 / 索引。
- 同步服务从单用户快照扩展到更细粒度增量同步。
- 团队模板、任务看板、工具调用可视化和 Soul Atrium 生态化。
